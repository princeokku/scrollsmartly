scrollsmartly

JavaScript Library

About

Webサイトにページ内スムーズスクロール機能を追加するためのJavaScriptライブラリです。単一ページのサイトなどの制作に役立ちます。

smartlyオブジェクトによる自由なユーザー設定

ライブラリの読み込み直後に生成されるsmartlyオブジェクトのプロパティおよびメソッドを用いて、デフォルト動作、スクロール速度などをサイト作成者が変更できます。

スクロールで「戻る」「進む」

スクロールした後にURLのハッシュ値を変更しますが、既存のスクロールライブラリには、ブラウザで履歴を前後した際に、URLのハッシュとページの表示位置が一致しないという問題がありました。 scrollsmartly.js では、履歴の前後に合わせて適切な位置にスクロールすることで、その問題を解決しています。

その他のライブラリに依存しない

jQueryなどの、本ライブラリ以外のJavaScriptライブラリとの依存関係がありません。ただし、Internet Exproler 6, 7などのHashChangeEventを実装していないブラウザでは、履歴の前後に伴うスクロールを行うことができません。旧バージョンのIEなどを含めた多くのブラウザで動作させるには、HashChangeEventを補完するライブラリを別途用意し、下記のように scrollsmartly.js より先にロードする必要があります。

    <script src='PATH/history.js'></script>
    <script src='PATH/scrollsmartly.js'></script>

HashChangeEventを補完できるライブラリはいくつか公開されていますが、動作確認を行っているのはHistory API JavaScript Library(© Dmitrii Pakhtinov)です。こちらの使用を推奨します。

Getting Started

最も簡単な使い方

smartly.allメソッドを呼び出すことで、href属性が#で始まる、通常はページ内リンクとして機能するA要素全てに対し自動的にスムーズスクロール機能を上書きします。

読み込み方の例

              <script src='PATH/scrollsmartly.js'></script>
              <script>
                smartly.all();
              </script>

API - Method

smartly.scroll([via1, ] [via2, … ] [to, ] [callback])

指定した要素へのスクロールを行います。経由する要素や、スクロールが完了した際に実行するコールバック関数を指定できます。

  • 引数(1): via1, via2, ...

    スクロール先に辿り着くまでに経由する要素を表すELEMENT Node、またはそれらの要素のid属性の文字列。省略した場合は、toで指定した最終目標に直接スクロールします。

  • 引数(2): to

    スクロール先の要素を表すELEMENT Node、またはスクロール先の要素のid属性の文字列。省略した場合は、smartly.homeElementにスクロールします。

  • 引数(3): callback

    スクロール完了後に実行するコールバック関数を指定します。

>> Example

smartly.delay([callback, ] [milliSeconds])

スクロールが終了するまでメソッドチェインの進行を停止します。milliSecondsミリ秒後に、メソッドチェインの進行を再開します。callbackが指定されていれば、再開前にそれを実行します。

  • 引数(1): function

    停止を終えた際に呼び出される関数を指定。

  • 引数(2): milliSeconds

    遅延する時間をミリ秒で指定。

>> Example

smartly.on(element [, target])

smartly.off(element)

指定した要素のクリックイベントに、smartly.scrollメソッドをイベントリスナーとして登録、または登録解除します。

  • 引数(1): element

  • 引数(2): target

smartly.replaceAnchor(element)

指定した要素がhref属性を持つ場合、ページ内リンクとしてのデフォルト動作を無効にし、代わりに、クリックした際に本来のリンク先を目標要素としてsmartly.scrollを実行します。

  • 引数(1): element

    スクロールボタンに入れ替える要素。

smartly.all()

全てのページ内リンクのデフォルト動作を無効にし、代わりに、本来のリンク先であるアンカーへのスムーズスクロールを起動するボタンにします。全てのリンクに対してsmartly.replaceAnchorメソッドを実行した場合と同様の動作です。

  • 引数: なし

smartly.set([properties])

smartlyオブジェクトの複数のプロパティをまとめて変更します。

  • 引数: Object

    smartlyオブジェクトのプロパティをkey、変更後の値をvalueとして持つオブジェクトを渡します。

API - Property

smartly.scrollingTo

  • 型: HTMLElement
  • 初期値: null

現在のスクロール先要素です。smartly.scrollメソッドが実行された際に、最終的なスクロール先となる要素が本プロパティに代入されます。
スクロールが完了したときに初期値であるnullが再度代入されます。スクロール中であるかどうかを判別したい場合は、下記のように処理します。

              if(smartly.scrolingTo !== null){
                //スクロール中であれば行う処理
              }

smartly.scrolledTo

  • 型: HTMLElement
  • 初期値: null

直前のスクロールで到達した要素です。smartly.scrollメソッドによるスクロールが完了した直後に、その際のスクロール先だった要素が代入されます。

smartly.homeElement

  • 型: HTMLElement
  • 初期値: document.documentElement

目標要素が指定されていないスクロール（例えばsmartly.scroll( )）は、全てこの要素へのスクロールとなります。特に何も指定していなければ、多くの場合ページの左上（X:0, Y:0）へのスクロールとなります。

smartly.position

  • 型: Stirng
  • 初期値: 'left top'

スクロール後、目標の要素がウィンドウ内のどこに位置するかを、キーワードかパーセントで指定します。 横の位置と縦の位置をスペースで区切り記述します。スペースで区切らずひとつだけ記述した場合、横位置と縦位置の両方に適用されます。 使用できるキーワードは、CSSのbackground-positionと同様のものです。

API - Event

smartlystart

  • 型: Object (HTMLEvent)
  • smartly.scrollメソッドが実行され、スクロールが始まる直前に発生します。

smartlyend

  • 型: Object (HTMLEvent)
  • smartly.scrollメソッドが実行され、スクロールが目標に到達した直後に発生します。
